# Лабораторна робота № 13

## Варіант №6
**тема:Медичні записи**

## Опис проєкту
Для виконання індивідуального завдання було обрано **Варіант №6: Медичні записи**. Програма імітує роботу інформаційної системи медичного центру, яка зчитує журнал реєстрації візитів пацієнтів із зовнішнього текстового файлу, проводить його повну валідацію та формує аналітичний підсумок.

## Що реалізовано в проєкті
У межах обов'язкової частини лабораторної роботи було повністю реалізовано такий функціонал:

* **Файлове введення (I/O):** Автоматичне зчитування сирих рядків із текстового файлу даних `medical-records.txt`, розміщеного у стандартній директорії ресурсів проєкту (`src/main/resources`). Порожні рядки під час зчитування ігноруються.
* **Попередня обробка та нормалізація тексту:** Використання базових операцій класу `String` (методів `trim()` та `replaceAll()`) для видалення випадкових або зайвих пробілів на початку, всередині та в кінці кожного текстового запису.
* **Складна валідація за шаблоном:** Розробка та застосування суворого регулярного виразу для перевірки відповідності кожного рядка встановленому формату журналу.
* **Вилучення даних (Capturing Groups):** Використання класів `Pattern` та `Matcher` для розбиття текстового рядка на логічні групи захоплення та їх подальшого вилучення (дата, час, ID пацієнта, спеціалізація лікаря, статус візиту).
* **Об'єктно-орієнтована модель (Domain Model):** Трансформація текстових токенів у типізовані об'єкти мови Java:
    * `RecordStatus` - безпечний перелік (`enum`) для обмеження допустимих станів візиту.
    * `MedicalRecord` - незмінний об'єкт класу (`immutable object`) для зберігання структурованих даних про один візит.
    * `MedicalReport` - об'єкт для збереження обчислених статистичних показників.
* **Ефективне формування результату:** Використання класу `StringBuilder` для поетапного збирання багаторядкового підсумкового аналітичного звіту без зайвої конкатенації рядків у циклі.
* **Модульне тестування (JUnit 5):** Написання автоматичних тестів для перевірки логіки нормалізації пробілів, розбору валідних даних, правильного підрахунку статистики та генерації винятків (`IllegalArgumentException`) у разі некоректного формату вхідного тексту.
